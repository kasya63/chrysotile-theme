Chrysotile — страницы ошибок и обслуживание
============================================

Что за чем (коротко)
--------------------
• README.md — НЕ нужен для работы сайта. Это только для людей/документации.
• wp-*.php в корне (wp-login.php, wp-config.php и т.д.) — это ядро WordPress.
  Для своих 404/503 их не создают и не правят «под дизайн».
• .htaccess — нужен WordPress’у в корне сайта (у вас уже есть). Для своей
  статической страницы 503 в него можно ДОБАВИТЬ пару строк — см. ниже.

Файлы, которые мы добавили
--------------------------
1) themes/cnn-child/404.php
   Срабатывает само: при несуществующем адресе WordPress отдаёт 404 и
   подключает этот шаблон (шапка/подвал темы, ваши стили).

2) themes/cnn-child/503-static.html
   Полностью автономная страница (без WordPress). Нужна, когда сайт «лёг»
   настолько, что PHP/БД не отвечают — тогда тема не загрузится.

   Как использовать на Apache (Local / типичный хостинг):
   • Скопируйте 503-static.html в КОРЕНЬ сайта (рядом с index.php), например
     как 503.html
   • В .htaccess в корне (ВНУРИ блока <IfModule mod_rewrite.c> или отдельно,
     как разрешает хостинг) добавьте:
       ErrorDocument 503 /503.html
   Путь должен быть URL-путь от корня сайта, не файловый.

   На nginx правило error_page задаётся в конфиге сервера — через панель
   хостинга или поддержку.

3) wp-content/maintenance.php
   Показывается ТОЛЬКО когда в корне сайта есть файл .maintenance
   (WordPress сам создаёт его на время обновления из админки).
   Если файл maintenance.php лежит в wp-content, WordPress подхватит его
   вместо стандартной серой страницы.

Итого: для «красивого 404» достаточно темы (404.php). Для «лежит всё»
— статика 503 + настройка веб-сервера; для обновлений из админки —
maintenance.php в wp-content.
